# Pygit Mini VCS

Учебный pet-проект на Python: мини-реализация системы контроля версий, похожей на Git.

Проект показывает, как внутри устроены базовые механизмы Git: инициализация репозитория, добавление файлов в индекс, создание объектов, коммиты, дерево файлов и просмотр истории.

## Описание проекта

`pygit` — это консольное приложение, которое реализует упрощённую логику Git.

В проекте есть собственная служебная директория `.pygit/`, где хранятся:

- объекты файлов;
- tree-объекты;
- commit-объекты;
- индекс;
- ссылка на текущую ветку `HEAD`.

Проект написан без использования готовых библиотек для работы с Git. Основная цель — разобраться, как устроены внутренние механизмы системы контроля версий.

## Основная функциональность

Реализованы команды:

- `init` — создание служебной директории `.pygit`;
- `add` — добавление файла в индекс;
- `commit` — создание коммита;
- `log` — просмотр истории коммитов;
- `write-tree` — построение tree-объекта из индекса.

## Что реализовано внутри

- хранение объектов в формате, похожем на Git;
- создание `blob`-объектов для содержимого файлов;
- создание `tree`-объектов для структуры директорий;
- создание `commit`-объектов с автором, сообщением и ссылкой на родителя;
- вычисление SHA-1 хеша объекта;
- сжатие объектов через `zlib`;
- хранение индекса в JSON-формате;
- обход истории коммитов через итератор;
- CLI-интерфейс для запуска команд из терминала;
- обработка ошибок при некорректных аргументах и отсутствии файлов.

## Как запустить

Запускать команды нужно из корня проекта.

### 1. Инициализировать mini-репозиторий

```bash
python -m lab3.pygit.pygit_commands init
```

После этого появится служебная директория `.pygit/`.

### 2. Создать тестовый файл

```bash
echo "Hello, pygit!" > hello.txt
```

### 3. Добавить файл в индекс

```bash
python -m lab3.pygit.pygit_commands add hello.txt
```

### 4. Создать коммит

```bash
python -m lab3.pygit.pygit_commands commit -m "Initial commit"
```

Команда выведет SHA-1 созданного коммита.

### 5. Посмотреть историю коммитов

```bash
python -m lab3.pygit.pygit_commands log
```

## Пример работы

python -m lab3.pygit.pygit_commands init

## Как устроено хранение объектов

Каждый объект сохраняется в `.pygit/objects/`.

Перед сохранением объект получает заголовок в формате:

```text
<type> <size>\0<data>
```

Затем:

1. считается SHA-1 хеш;
2. объект сжимается через `zlib`;
3. сохраняется в директорию `.pygit/objects/`.

Например, объект с хешем:

```text
abcdef123...
```

будет храниться по пути:

```text
.pygit/objects/ab/cdef123...
```

## Типы объектов

### Blob

`Blob` хранит содержимое файла.

### Tree

`Tree` хранит структуру директории: имена файлов, режимы доступа и SHA объектов.

### Commit

`Commit` хранит:

- SHA дерева;
- SHA родительского коммита;
- автора;
- сообщение коммита.

## Индекс

Индекс хранится в файле:

```text
.pygit/index
```

Формат индекса — JSON:

```json
[
  ["hello.txt", "sha1_hash", 33188]
]
```

Индекс используется как staging area: сначала файлы добавляются через `add`, потом из индекса строится tree-объект и создаётся commit.

## Цель проекта

Цель проекта — изучить внутреннюю логику Git на практике:

- как Git хранит объекты;
- как работает индекс;
- как создаются коммиты;
- как устроена связь commit → tree → blob;
- как можно реализовать CLI-инструмент на Python.

Проект выполнен в учебных целях.
